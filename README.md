# 3D Washing Machine Model

<br>

## 📖 Description

Interactive 3D OpenGL model of a washing machine with manually defined geometry and realistic lighting.
Supports animations, camera control, and object interaction, including door opening, drum rotation, and control knob movement.

<br>

---

## ✨ Features

- Scene Interaction:
  - Translation (objects, lights, and camera)
  - Rotation (objects and camera; lights are fixed in rotation)
- Lighting Control:
  - Toggle global lighting
  - Toggle individual lights
  - Adjust light intensity
- Animations:
    - Door opening/closing
    - Detergent drawer extension
    - Display activation
- Input System: Keyboard-based controls
- Console Output:
    - Displays full scene and object info at runtime, including:
        - Shortcuts
        - Object coordinates
        - Light coordinates and intensity
        - Loaded textures and their indices
- Texturing: Manual assignment required

## 🏗️ Project Structure
```
3d-washing-machine-model/
│
├── build/            # Build output generated by CMake (not part of source code)
|
├── external/         # Third-party dependencies
│   ├── glfw/         # Precompiled GLFW (headers, import library, DLL)
│   ├── mingw/        # Required MinGW runtime DLLs
│   └── stb/          # stb_image (texture loader)
│
├── inc/              # Header files
│   ├── config/       # Global configuration and constants
│   ├── core/         # Input handling system
│   ├── model/        # Washing machine components (3D objects)
│   ├── scene/        # Scene management, rendering, lighting, camera
│   └── utils/        # Math utilities, textures, console tools
│
├── src/              # Source files (implementation)
│   ├── config/
│   ├── core/
│   ├── model/
│   ├── scene/
│   └── utils/
│
├── textures/         # Scene textures (floor, walls)
│
└── CMakeLists.txt    # Build configuration file
```

<br>

### Notes

- The `build/` directory is generated by **CMake** and is not part of the source code.
- External dependencies are stored locally to ensure **easy setup without package managers**.

<br>

## 🧩 Architecture Overview

The project follows a modular structure inspired by a simplified 3D engine architecture.

- **Model layer** – individual washing machine components (door, drum, tank, etc.), each with its own transformation and behavior
- **Scene layer** – rendering pipeline, camera system, projection, and lighting
- **Core layer** – input handling and application control logic
- **Utils layer** – math operations, texture loading, and helper utilities

Each object is responsible for its own transformation and rendering, enabling independent animation and interaction within the scene.

## 🚀 Getting Started

### Requirements
- C++ compiler
- CMake
- GLFW - 3.3.8 (included in the project as DLL, import library, and headers; included files are provided for **MinGW-w64** usage)

> Recommended: On Windows, use **MSYS2** (includes MinGW-w64, CMake, Ninja, and other necessary tools) to simplify setup.

<br>

### Setup
Clone the repository:

```
git clone https://github.com/krzysztof-szczepanik/3d-washing-machine-model.git
```

<br>

### Build
Using terminal, in the **root of the cloned repository**, run:

```
mkdir build
cd build
cmake ..
cmake --build .
```

> Note: If you are using **MSYS2 terminal**, make sure to use the **MSYS2 MINGW64 terminal**.  
> The default **MSYS2 MSYS terminal** may not work correctly with `cmake`.

<br>

### Run
Run the application:

```
./3DWashingMachineModel.exe
```

> Note: The required DLLs for **MinGW-w64** are already included in `external/mingw/bin`.
> 
> If you are using a different MinGW installation, replace them with the DLLs from your own MinGW bin folder to avoid potential runtime errors:
> - libgcc_s_seh-1.dll
> - libstdc++-6.dll
> - libwinpthread-1.dll

<br>

### Notes
- The project is configured primarily for **MinGW-w64 on Windows**
- If the application does not start, ensure that required `.dll` files are present in the build directory
- On other platforms, GLFW linking may require manual configuration

<br>

### Tested Environment
- **MSYS2** with **MinGW-w64**
- GLFW DLLs provided by the project
